Reglas requeridas para utilizar el analizador de código.\n 1.Debe ingresar código que compile.\n 2.Los ; de las instrucciones deben estar pegados a las mismas: correcto int x=0; incorrecto int x=0 ;. \n 3.Los switchs no están permitidos. \n 4.No se permite declarar/inicializar en la misma línea: correcto int x=0; int y=0; incorrecto int x,y =0; \n 5.Debe utilizar llaves en sus clases, metodos, ciclos, if y else, y estas deben estar pegadas a la instrcción. \n 6.Los métodos: \n Deben ser solamente: \n \t public \n \t private \n \t static \n Y retornar: \n \t void \n \t short \n \t int \n \t float \n \t double \n \t long \n \t char \n \t bool \n \t string \n Se permite el uso de: \n \t std \n \t unsigned \n \t signed \n 8.Las clases solamente pueden ser o tener: \n \t public \n \t private \n \t static; \n \t class
